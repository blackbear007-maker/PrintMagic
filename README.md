# ImageTool

瀏覽器端的圖片轉換與印刷適合度檢查工具。專為「AI 生成圖想拿去印刷」的新手設計：無後端、無付費 API、所有處理都在本機完成。

## 功能

- **印刷場景選單**：選擇「社群分享 / 明信片 / A4 海報 / A3 海報 / 名片 / 貼紙」，自動設定 DPI、出血與格式。
- **印刷成功率評分**：根據圖片實際像素、亮度、對比、飽和度、銳利度、格式，計算誠實的 0–100 分預估分數。
- **AI 圖片放大**：使用 Real-CUGAN 模型進行 AI 超解析度放大，首次使用需下載模型（約 2.6MB），之後可永久離線使用。AI 放大失敗時自動回退到 Lanczos 插值。
- **印刷優化功能**：
  - **總墨量控制**：自動檢測並修正超過 320% 總墨量的像素，避免油墨過厚導致乾燥問題。
  - **DPI 自動檢測**：自動計算圖片 DPI，低於 300 DPI 時自動放大至標準印刷解析度。
  - **印刷專用銳化**：AI 放大後自動應用 Unsharp Mask 銳化（Amount 150%, Radius 1px, Threshold 3），提升印刷清晰度。
- **PDF 匯出**：含裁切標記、出血區域、印刷廠資訊，首次使用需下載 jsPDF 庫（約 200KB），之後可永久離線使用。
- **批次處理**：一次選擇多張圖片，依序自動處理與下載。
- **完全離線**：`index.html` 已內嵌所有程式與 Worker，AI 模型和 jsPDF 首次下載後可永久離線使用。
- **自動測試**：`tests.html` 測試核心功能；`print-test.html` 收集真實印刷結果。
- **無 VIP、無浮水印、無 AI 誇大宣傳**。

## 使用方式

最簡單的方式：直接開啟 `index.html`，拖圖片進去即可。

### AI 放大功能

首次使用 AI 放大功能時：
1. 點擊「2x 放大」或「4x 放大」按鈕
2. 系統會提示需要下載 AI 模型（約 2.6MB）
3. 確認下載後，模型會緩存到瀏覽器的 IndexedDB
4. 下載完成後可永久離線使用，無需再次下載

### PDF 匯出功能

首次使用 PDF 匯出功能時：
1. 選擇 PDF 格式匯出
2. 系統會提示需要下載 jsPDF 庫（約 200KB）
3. 確認下載後，庫會緩存到瀏覽器的 IndexedDB
4. 下載完成後可永久離線使用，無需再次下載

PDF 匯出自動執行印刷優化：
- DPI 檢測和自動放大（低於 300 DPI 時）
- 總墨量控制（超過 320% 時自動修正）
- 印刷專用銳化

PDF 包含：
- 裁切標記（可選）
- 精確尺寸（mm）
- 出血區域
- 印刷廠資訊

### 印刷優化（自動執行）

所有匯出格式（PNG、PDF）都會自動執行印刷優化：
1. **DPI 檢測**：計算圖片實際 DPI
2. **自動放大**：低於 300 DPI 時使用 AI 放大至標準解析度
3. **總墨量控制**：檢測並修正超過 320% 的像素
4. **印刷銳化**：應用 Unsharp Mask 提升清晰度

若要啟動本機伺服器：

```bash
npm start
# 或
python -m http.server 8080
```

然後開啟 `http://localhost:8080/index.html`。

## 專案結構

- `index.html` — 主要入口，完全離線單檔版（內嵌所有 JS 和 Worker）
- `image-core.js` — 核心 API（匯出、放大、設定、印刷評分）- 已內嵌到 index.html
- `image-worker.js` — Web Worker 像素運算 - 已內嵌到 index.html
- `tests.html` — 瀏覽器自動測試
- `print-test.html` — 真實印刷測試迴圈，收集成功/失敗案例
- `print-shop-spec.md` — 給印刷店看的輸出規格
- `manifest.json` — PWA manifest
- `sw.js` — Service Worker 離線快取
- `tools/` — 開發用腳本
